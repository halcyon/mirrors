The REPL



REPL stands for read–eval­u­ate-print loop. It’s pro­nounced rep­ple. Racket’s REPL is an inter­ac­tive prompt where you can type Racket expres­sions and see the results. The REPL is a quick way to run small cal­cu­la­tions.



The REPL is avail­able two ways:

  1. 

    In your ter­mi­nal win­dow, by launch­ing racket:

    
    ~ : racket

    42
    -> (current-milliseconds)
    -> 1489623272609
    -> (define xyzzy (random 100))
    -> xyzzy
    83

-> (* 6 7)

-> xyzzy

    

    You can spec­ify a dif­fer­ent lan­guage for this repl with the -I flag:

    
    ~ : racket -I br
    Welcome to Racket v6.8.0.2.
    -> (define-macro (foo THIS) #'42)
    -> (foo "you")
    42

  2. 

    You can also access the REPL in DrRacket, by using the prompt in the inter­ac­tions panel:

    
    Welcome to DrRacket, version 6.8.0.2 [3m].
    Language: racket [custom]; memory limit: 3000 MB.
    > (* 6 7)
    42
    >
    1
2
3
4
5
    Welcome to DrRacket, version 6.8.0.2 [3m].
Language: racket [custom]; memory limit: 3000 MB.
> (* 6 7)
42
>
    

    Unlike the com­mand-line REPL, the DrRacket REPL adopts the lan­guage spec­i­fied by the #lang line at the top of the def­i­n­i­tions panel. DrRacket also uses the REPL itself to print out­put from the pro­gram.

    

    The DrRacket REPL lets you use vari­ables and func­tions from the source file in the def­i­n­i­tions win­dow (even if they’re not made avail­able through provide). So if this is the pro­gram:

    
    (define foo "bar")
    (define (repeat str) (string-append str str))
    1
2
    (define foo "bar")
(define (repeat str) (string-append str str))
    

    After you run the file (to update the def­i­n­i­tions), you can use them at the DrRacket REPL:

    
    Welcome to DrRacket, version 6.8.0.2 [3m].
    Language: br [custom]; memory limit: 3000 MB.
    > foo
    "bar"
    > (repeat foo)
    "barbar"
    >
    1
2
3
4
5
6
7
    Welcome to DrRacket, version 6.8.0.2 [3m].
Language: br [custom]; memory limit: 3000 MB.
> foo
"bar"
> (repeat foo)
"barbar"
>
    

    Tex­tual out­put is just the begin­ning of the DrRacket REPL. It can dis­play a range of data for­mats, includ­ing images and inter­ac­tive plots.



Mak­ing the REPL avail­able in a lan­guage



Cus­tom lan­guages can acti­vate the REPL by pro­vid­ing #%top-interaction from the expander. (br/quicklang han­dles this for you, so lan­guages made with br/quicklang auto­mat­i­cally have REPL sup­port.)



More intri­cate REPL hack­ing is pos­si­ble by adjust­ing the run­time con­fig­u­ra­tion of the lan­guage.
