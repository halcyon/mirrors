Setting the Mac OS PATH



What is path?



PATH is a sys­tem-level vari­able that holds a list of direc­to­ries. When you enter a com­mand in the ter­mi­nal, it’s short­hand for a pro­gram with the same name. The sys­tem looks in each of the PATH direc­to­ries for the pro­gram cor­re­spond­ing to the com­mand. When it finds a match­ing pro­gram, it runs it. If it doesn’t find a match, it raises an error.



To see the PATH on your sys­tem, open the ter­mi­nal win­dow and enter this com­mand (where sh$ rep­re­sents the ter­mi­nal prompt, which will be dif­fer­ent on your sys­tem):


sh$ echo $PATH
1
sh$ echo $PATH


Type return to com­plete the com­mand. The result will be a list of direc­to­ries sep­a­rated by : that looks like this:


/usr/bin:/bin:/usr/sbin:/usr/local/bin
1
/usr/bin:/bin:/usr/sbin:/usr/local/bin


The exact list may be dif­fer­ent on your sys­tem. Now try this:


sh$ which echo
/bin/echo
1
2
sh$ which echo
/bin/echo


This tells us that the echo com­mand (which prints a value to the ter­mi­nal) actu­ally invokes the pro­gram located at /bin/echo. The ter­mi­nal finds this pro­gram by look­ing for echo in each of the PATH direc­to­ries, one of which is /bin.



We can always invoke a com­mand by using its full path rather than the short­hand:


sh$ /bin/echo $PATH
/usr/bin:/bin:/usr/sbin:/sbin:/usr/local/bin
1
2
sh$ /bin/echo $PATH
/usr/bin:/bin:/usr/sbin:/sbin:/usr/local/bin


In this case, the ter­mi­nal doesn’t look in the PATH direc­to­ries for echo, because we’ve already pro­vided a full path to echo.



The which com­mand shows us the full path cor­re­spond­ing to a com­mand. We can even do this:


sh$ which which
/usr/bin/which
1
2
sh$ which which
/usr/bin/which


If we enter a com­mand name that doesn’t exist in any PATH direc­tory, we get an error:


sh$ racket
-bash: racket: command not found
1
2
sh$ racket
-bash: racket: command not found


Our goal is to add the direc­tory con­tain­ing racket to our PATH. Then the ter­mi­nal will be able to find it.



Updat­ing path



First, check that your Racket instal­la­tion works by going into your new Racket direc­tory and launch­ing DrRacket. If DrRacket works, then your Racket instal­la­tion is sound.



Next, check that you know the full path to the bin sub­di­rec­tory of your new Racket direc­tory. If you put this direc­tory in Applications as rec­om­mended, the path will be:


/Applications/Racket v6.8/bin
1
/Applications/Racket v6.8/bin


If you put it else­where, adjust the next instruc­tion accord­ingly.



You can test this path from the ter­mi­nal: add /racket to the path and enter it as a com­mand. This should start Racket (include the sur­round­ing dou­ble quotes around the whole thing):


sh$ "/Applications/Racket v6.8/bin/racket"
Welcome to Racket v.6.8.
>
1
2
3
sh$ "/Applications/Racket v6.8/bin/racket"
Welcome to Racket v.6.8.
>


If this doesn’t work, there’s an error in your Racket path.



If it does work, you need to add this path to the "/etc/paths" file on your sys­tem, which con­trols the sys­tem PATH vari­able. You can do this with the fol­low­ing one-line ter­mi­nal com­mand, which will ask you for your pass­word (replace /path/to/racket/bin with your actual path, again includ­ing the sur­round­ing dou­ble quotes):


sh$ sudo sh -c 'echo "/path/to/racket/bin" >> /etc/paths'
1
sh$ sudo sh -c 'echo "/path/to/racket/bin" >> /etc/paths'


To check that your PATH has been updated cor­rectly, open a new ter­mi­nal win­dow and try echo $PATH again:


sh$ echo $PATH
/usr/bin:/bin:/usr/sbin:/usr/local/bin:/path/to/racket/bin
1
2
sh$ echo $PATH
/usr/bin:/bin:/usr/sbin:/usr/local/bin:/path/to/racket/bin


This time, you should see Racket’s direc­tory at the end of the PATH list of direc­to­ries.



If so, you can now try which racket:


sh$ which racket
/path/to/racket/bin/racket
1
2
sh$ which racket
/path/to/racket/bin/racket


If that works, then you’re home free. Type racket and you should see some­thing like this:


sh$ racket
Welcome to Racket v.6.8.
>
1
2
3
sh$ racket
Welcome to Racket v.6.8.
>


If so, all is well. The > is the com­mand prompt for Racket. Type ctrl+D to exit Racket.
